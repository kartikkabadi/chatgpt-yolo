# Screenshot alt text

Alt text for the five product screenshots in `docs/assets/`, written for use in
the README and other launch surfaces. Each description is concrete, avoids
implying capabilities the product does not have, and reflects that the states
are a staged demonstration (see `README.md` for the full disclosure).

## `screenshot-queue.webp`

> The YOLO for ChatGPT popup docked beside a ChatGPT conversation. The popup
> shows the automation running in Balanced mode with five queued instructions
> for auditing a repository: "Review the implementation for concrete edge cases"
> is sending, followed by four queued steps to run validation, fix failures,
> review the diff, and summarize. Each item shows its order number and state,
> with reorder, edit, and remove controls.

## `screenshot-command-palette.webp`

> The YOLO command palette open over a ChatGPT conversation. A search field
> reads "Type a command" and the list shows YOLO commands with descriptions:
> /goal and /loop for bounded workflows, and /plan, /review, /fix, /handoff, and
> /continue prompt shortcuts. A footer shows keyboard navigation hints.

## `screenshot-workflow.webp`

> An active YOLO Goal workflow shown over a ChatGPT conversation. A status panel
> lists the workflow as Goal, running, with the objective to audit a repository,
> iteration 2 of 5, one item sending with four queued, an hourly limit of 6 of
> 30 used, draft protection on, and the last action delivered 12 seconds ago. A
> workflow bar at the composer shows the objective with Pause, Edit, and Stop
> controls.

## `screenshot-github-workflow.webp`

> A ChatGPT conversation working on a connected GitHub repository, with a small
> "Optional · GitHub app connected to ChatGPT" label. A note clarifies that
> GitHub connects to ChatGPT, not to YOLO. The YOLO popup manages four queued
> follow-up prompts — review the changes for regressions, run validation, fix
> findings, and summarize the final state — while GitHub provides repository
> context to ChatGPT separately.

## `screenshot-settings.webp`

> The YOLO Advanced settings page. A sidebar lists sections including Overview,
> Queue, Approvals, Recovery, and Safety. The Overview shows the Balanced
> operating profile with automation enabled. The Approvals section is off by
> default with a conservative "non-sensitive confirmations only" policy and
> per-hour limits, reflecting local, opt-in controls.
